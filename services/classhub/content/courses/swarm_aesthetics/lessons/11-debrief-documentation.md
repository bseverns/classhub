## At a glance
- **Big question:** How do you make an ephemeral event teachable?
- **Outputs:** documentation bundle draft + peer edits
- **In-class mode:** edit lab + critique of documentation

## Learning goals
Students will:
- craft a coherent documentation narrative
- connect evidence to claims
- prepare work for final critique/publishing

## Before class (instructor)
- Prepare an example “documentation bundle” (even a simple one).
- Print `templates/DOCUMENTATION_BUNDLE_CHECKLIST.md`.

## Materials
- laptops or paper layout tools
- access to shared documentation folder

## In-class plan (150 min)
1) **Debrief circle (20)**  
   Each team shares one surprising ethical/legibility lesson.

2) **Bundle assembly sprint (55)**  
   Compile: maps, recipe, roles, signage, scripts, traces, postmortem, run-it-again guide.

3) **Peer edit (45)**  
   Teams swap bundles. Feedback focuses on:
   - what’s missing to run it again
   - what’s unclear or ethically risky
   - what evidence is needed

4) **Revision plan (20)**  
   Teams write a concrete checklist for final week.

5) **Close (10)**

## After class / homework
- Submit **Documentation Bundle Draft**.
- **Micro #11:** “letter to future runners” (or fold into postmortem).
